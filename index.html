Identifying Usability Problems

This is the link to the Original Website. You can also see the below embedded website, and the screenshot of it. This website is extremely simple and has a style from early 2000s, but it has major problems with the UI placement especially when changing the screen size, which is the reason why I decided to redesign thispage.




Problems With the Current Website Design

  • There is a youtube video link on the website, but the user has to leave the webpage to view it. Since we want to use less internet usage in the case when the user is on phone, we want to just show the youtube on the page when the user is on PC (the screen size is at least the size of standard PC).
  • Font size for the language selector is small
  • Navigation bar / menu does not collapse when the screen size changes
  • Horizontal scroll is required in some cases to view the full page in some cases.
  • The Information section is located towards the bottom of the page. Important information should be placed towards the top so that the user can easily view it.
  • The title scrolls from right to left, which seems to be unnecessary.
  • Website has a heart-shape cursor, but this is making things even harder to read.
  • The opacity of background image should be changed to make it easier to read the content.
  • When the screen size changes, the main text takes up the entire width of the screen, making it hard to read.


Accessibility

I used WebAIM WAVE to detect possible accessibility problems. WebAIM WAVE detected quite a lot of contrast problems on the website. I totally agree with this, and as I mentioned in the previous section, I believe that changing the text color and background opacity will improve the overall readability. Additionally, it alerts about the marquee element (scrolling content that the user can not stop). I believe that this is also decreasing the readability of the page. Lastly, layout table is being used in the website. This is some feature that should not be used in HTML5 according to WebAIM, which I agree with since it decreases the readability of the page if the screen size changes.

Style Guides for My Design

Since the project is mainly built using bootstrap, there are no customized buttons, and since we are using the same background as the original website, there are no changes to that. Below are the simple guidelines I have. For this reason, there are not many content in the figma embedding below. As a reference, this is the link to the design guides.

Pleae also view the hifi-prototype Figma in the below section as well since that also contains style guides.
  • font-family: Georgia, 'Times New Roman', Times, serif
  • Navigation bar font size: large
  • URL Link Color: Blue when not hovered, Red(#f00) and underlined when hovered. Does not change the text color after visiting the website
  • All bootstrap containers have width of 75%
  • Youtube thumbnail should only be visible when the screen is larger than 767px. Otherwise, just show the original png
